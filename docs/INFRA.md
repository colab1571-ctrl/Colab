# Infrastructure Setup Guide — Colab

> Generated by Phase 3b of the unified-development pipeline. Every external service the locked stack depends on, with sign-up instructions, scopes, OAuth posture, and env-var keys.

**Order**: complete services in this order — earlier services unblock later ones.

---

## 1. AWS (root account + first IAM)

**Why**: Hosts EKS (12+ FastAPI services), RDS Postgres, ElastiCache Redis, S3, CloudFront, Amazon MQ, SES, SNS Mobile Push, Secrets Manager, API Gateway, ACM, Route 53.

**Steps**:
1. Sign up at https://aws.amazon.com — billing requires phone OTP + credit card + ID verification (manual).
2. Enable MFA on the root account, then immediately stop using it.
3. Create an IAM admin user + group; install AWS CLI v2; run `aws configure --profile colab-admin`.
4. Run `terraform/bootstrap.sh` (writes a Terraform state S3 bucket + DynamoDB lock table).
5. Run `terraform -chdir=terraform/envs/prod apply` — provisions VPC, EKS, RDS, Redis, S3, MQ, SES, IAM roles.
6. Add the admin user's access keys to your local keyring (do **not** commit to .env in repo).
7. After IAM-roles-for-service-accounts (IRSA) is up, all pods authenticate through IRSA — no static AWS keys in .env in production.

**Region**: `us-east-1` (primary). All resources tagged `Project=colab`, `Env=prod|staging|dev`.

**Env-vars**: `AWS_REGION`, `AWS_ACCOUNT_ID`. Service-specific names below.

## 2. Domain name + DNS

**Why**: marketing site, app domain, API gateway, OAuth redirect URIs.

**Steps**:
1. Register a domain (Route 53 directly, or Cloudflare/Namecheap and delegate NS to Route 53 hosted zone).
2. Create Route 53 hosted zone via Terraform (`terraform/modules/dns`).
3. Request an ACM certificate (wildcard `*.<domain>` + apex) in `us-east-1`.

**Env-vars**: `APP_DOMAIN`, `MARKETING_DOMAIN`, `ADMIN_DOMAIN`, `API_DOMAIN`.

---

## 3. GitHub repo + Actions OIDC

**Why**: source control (already), CI/CD, OIDC-based deploy auth to AWS.

**Steps**:
1. Repo: `https://github.com/colab1571-ctrl/Colab` (already configured).
2. Create branch protection on `main`: require PR, require status checks, require linear history.
3. Configure the `github-actions` IAM role with OIDC trust (`terraform/modules/github-oidc`).
4. Add `AWS_DEPLOY_ROLE_ARN` to repo Variables (not Secrets — it's not sensitive).
5. Add other secrets (vendor API keys) as repo Secrets — see §15 .env mapping.

## 4. Apple Developer Program

**Why**: iOS app distribution, Sign in with Apple, Apple IAP (gated to RevenueCat).

**Steps** (manual — Apple ID 2FA required on your physical device):
1. Enroll at https://developer.apple.com/programs ($99/yr individual; D-U-N-S required for organization).
2. Create an App ID for `com.<your-org>.colab` with capabilities: Push Notifications, Sign in with Apple, Associated Domains, In-App Purchase.
3. Create an APNs Auth Key (.p8) — used by AWS SNS Mobile Push.
4. Create a Sign in with Apple Services ID (different from the App ID) — used for web Sign in with Apple if needed.
5. Generate an App Store Connect API Key (for RevenueCat + Fastlane CI).
6. Add the team to App Store Connect; create the app record.

**Env-vars** (server-side): `APPLE_TEAM_ID`, `APPLE_BUNDLE_ID`, `APPLE_KEY_ID` (APNs), `APPLE_PRIVATE_KEY` (.p8 contents).
**Env-vars** (client-side): `APPLE_SIGN_IN_CLIENT_ID` (the Services ID).

---

## 5. Google Cloud Console project

**Why**: Google Sign-In OAuth, YouTube Data API (profile linking), Google Meet API (meeting creation), Google Calendar API.

**Steps**:
1. Create a project at https://console.cloud.google.com (Google account 2FA required).
2. Enable APIs: People API, YouTube Data API v3, Google Calendar API, Google Meet API.
3. Configure OAuth consent screen (external, 18+, brand name + verified logo when known).
4. Create OAuth 2.0 Client IDs:
   - `iOS` client (bundle id matches Apple)
   - `Android` client (package name + SHA-1 of release key)
   - `Web` client (consumer-web + marketing redirect URIs)
   - `Server` client (server-to-server flows for YouTube + Calendar)
5. Generate a service account for Recall.ai bot to schedule meetings via Calendar API.

**Env-vars**: `GOOGLE_CLIENT_ID_IOS`, `GOOGLE_CLIENT_ID_ANDROID`, `GOOGLE_CLIENT_ID_WEB`, `GOOGLE_CLIENT_SECRET_WEB`, `GOOGLE_SERVICE_ACCOUNT_JSON`.

## 6. Google Play Console

**Why**: Android app distribution, Google Play Billing (gated to RevenueCat).

**Steps**:
1. Pay $25 one-time fee; KYC verification (manual).
2. Create the app record (package `com.<your-org>.colab`).
3. Generate a service account JSON for RevenueCat + Fastlane CI to upload AABs and manage IAP products.
4. Configure FCM credentials for push (Project Settings → Cloud Messaging — copy Sender ID + Server Key).

**Env-vars**: `PLAY_PACKAGE_NAME`, `PLAY_SERVICE_ACCOUNT_JSON`, `FCM_SENDER_ID`, `FCM_SERVER_KEY` (also wired to AWS SNS Mobile Push).

---

## 7. Persona (selfie + liveness)

**Why**: Identity verification for the Valid Profile Badge.

**Steps**:
1. Sign up at https://withpersona.com (free sandbox; production billing requires KYC on your business).
2. Create a Verification Template: Selfie + Liveness (head-turn workflow). Configure under-18 face-age signal → manual review.
3. Generate an Inquiry API key + Webhook signing secret.
4. Configure webhook URL → `https://api.<domain>/webhooks/persona/inquiry`.

**Env-vars**: `PERSONA_API_KEY`, `PERSONA_TEMPLATE_ID`, `PERSONA_WEBHOOK_SECRET`.

---

## 8. Stripe (web payments, tax, partner payouts, credit purchases)

**Why**: Web subscriptions, Stripe Tax, credit-pack one-time purchases, partner-payout (when ads ship).

**Steps**:
1. Activate Stripe account (business legal entity + bank account + tax ID — manual KYC).
2. Enable Stripe Tax for US, CA, AU, NZ (IN handled separately).
3. Create Products + Prices for Premium-month, Premium-year, Pro-month, Pro-year, Credit-pack-small, Credit-pack-medium, Credit-pack-large (values blank initially — admin console fills via dashboard).
4. Configure webhook → `https://api.<domain>/webhooks/stripe`.
5. Webhook events to subscribe: `customer.*`, `customer.subscription.*`, `invoice.*`, `payment_intent.*`, `charge.dispute.*`, `tax.*`.

**Env-vars**: `STRIPE_SECRET_KEY`, `STRIPE_PUBLISHABLE_KEY`, `STRIPE_WEBHOOK_SECRET`, `STRIPE_PRICE_ID_PREMIUM_MONTH`, `STRIPE_PRICE_ID_PREMIUM_YEAR`, etc.

## 9. RevenueCat (mobile IAP + cross-platform entitlement)

**Why**: iOS Apple IAP + Android Play Billing receipt validation, cross-platform entitlement parity, store webhook reconciliation, Stripe linking for web.

**Steps**:
1. Sign up at https://app.revenuecat.com (Google SSO available).
2. Create the iOS app (bundle id) + Android app (package).
3. Upload the App Store Connect API key (from §4) + Play Service Account JSON (from §6).
4. Define entitlements: `premium`, `pro`. Define products mapped to App Store + Play Console product IDs.
5. Link Stripe as a non-store backend integration → enables web checkout to grant the same entitlements.
6. Configure webhook → `https://api.<domain>/webhooks/revenuecat`.

**Env-vars**: `REVENUECAT_API_KEY_IOS` (public, RN), `REVENUECAT_API_KEY_ANDROID` (public, RN), `REVENUECAT_SECRET_KEY` (server-side), `REVENUECAT_WEBHOOK_SECRET`.

## Per-service "what I can drive vs you must do"

| Service | Sign-up automation | Daily ops automation |
|---|---|---|
| AWS | **You** (KYC) | I (Terraform + GitHub Actions) |
| Apple Developer / App Store Connect | **You** (Apple ID + $99 + D-U-N-S) | I (Fastlane via CI once keys are stored) |
| Google Play | **You** ($25 + KYC) | I (Fastlane via CI once service account JSON is stored) |
| Google Cloud | **You** (account + payment) | I (Terraform google provider) |
| Stripe | **You** (KYC + bank) | I (Stripe API via CI for Product/Price config) |
| RevenueCat | **You** (sign-up + key uploads) | I (configure via API once keys are in) |
| Persona | **You** (sign-up + KYB for prod) | I (template via API) |
| OpenAI / Replicate / Mapbox / Sentry / PostHog / Recall.ai | **You** (sign-up + payment) | I (programmatic config via API) |
| Meta / Spotify | **You** (sign-up + app review) | I cannot drive app review; that requires your screencast |
| EAS | **You** (sign-up) | I (eas.json config + CI) |
| Domain DNS | **You** (registrar) | I (Route 53 records via Terraform) |

## Order of execution (recommended)

1. AWS root account → IAM admin user → bootstrap Terraform state
2. GitHub Actions OIDC role
3. Domain + Route 53 + ACM cert
4. Terraform apply: VPC + EKS + RDS + Redis + S3 + MQ + SES + IAM
5. OpenAI + Replicate + Mapbox + Sentry + PostHog + Recall.ai (parallel — sign-up via OAuth where supported)
6. Persona (sandbox first)
7. Apple Developer + Google Cloud + Google Play (parallel — these block iOS/Android builds)
8. Stripe + RevenueCat (parallel — Stripe must complete before RevenueCat web link)
9. Meta + Spotify dev apps (deferred until OAuth integration phase — they require app review, which takes 2–6 weeks)
10. EAS account + iOS/Android credentials upload

## Secrets storage

- Local dev: `.env` files (gitignored). Reference values from `.env.example`.
- CI/CD: GitHub Actions Secrets (per the .env map below).
- Production: AWS Secrets Manager + Parameter Store. Pods read via IRSA-bound IAM roles.

`.env` is in `.gitignore`. `.env.example` is committed and serves as the canonical list.

See `.env.example` for the full variable inventory.
